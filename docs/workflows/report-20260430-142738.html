1. Code Quality & Standards
1.1 Critical: Typos in Core Framework Decorators
CRITICAL — @Injectable() mispelled as @Injectable() in ALL NestJS services
Affects: central-hub-api, customer-portal-api, vendor-api-server | File: */app.service.ts
// INCORCT (all 3 API services): import { Injectable } from '@nestjs/common'; // TYPO: should be Injectable @Injectable() // WON'T WORK - decorator won't register the class export class AppService { ... } // CORRECT: import { Injectable } from '@nestjs/common'; @Injectable() export class AppService { ... }

Impact: NestJS dependency injection will fail. The @Injectable() decorator is misspelled as @Injectable() — these are two different tokens. This will cause runtime errors.

1.2 High: HTML Attribute Typos in All Next.js Apps
HIGH — rel="noreferrer" typo in ALL dashboard pages
Affects: 3 dashboards, 1 public website | 18+ occurrences across files
// INCORCT: // TYPO: should be noreferrer // CORRECT:

Impact: The rel attribute is ignored by browsers. Security feature (noopener) is not enforced, creating potential tab-napping vulnerabilities.

2. Architecture & System Design
2.1 High: Missing @ethio-connect/infrastructure Project
HIGH — Workflows reference non-existent infrastructure project
Files: ci.yml, release.yml, promote.yml, rollback.yml (removed but originally referenced)

The CI/CD workflows originally referenced @ethio-connect/infrastructure project, libs/infrastructure/scripts/*.mjs, and environment YAML files — none of which exist. This would cause all deploy/promote/rollback workflows to fail.

Resolution: Either create the infrastructure project with proper structure, or remove all infrastructure dependencies from workflows (which has now been done).
2.2 High: No Shared API Modules Used
HIGH — @ethio-connect/api-common and @ethio-connect/api-features are empty
Files: libs/api-common/src/lib/api-common.module.ts, libs/api-features/src/lib/api-features.module.ts
// CURRENT STATE (empty modules): @Module({ controllers: [], providers: [], exports: [] }) export class EthioConnectApiCommonModule {} // EXPECTED (example): @Module({ controllers: [HealthController], providers: [AuthService, PrismaService], exports: [AuthService] }) export class EthioConnectApiCommonModule {}

Impact: No shared authentication, database access, or common utilities exist across the 3 API services. Each API will duplicate code.

2.3 Medium: No Clear API->Frontend Integration Pattern
MEDIUM — No API client, no data fetching pattern defined

There's no defined pattern for how Next.js frontends consume NestJS APIs. No Swagger/OpenAPI spec, no generated clients, no tRPC or REST client setup.

2.4 Medium: Monorepo Module Boundary Violations Possible
MEDIUM — ESLint module boundary rules defined but not enforced in CI

The eslint.config.mjs defines tags (scope:central-hub, type:api, etc.) but the CI workflow doesn't run lint as a blocking step for all projects.

2.5 Low: No Database/ORM Layer Defined

For a production app with customers and vendors, there's no Prisma, TypeORM, or any data layer defined. This is expected in scaffold state but must be prioritized.

4. Security Audit
4.1 Critical: CORS Not Configured on NestJS APIs
CRITICAL — No CORS, Helmet, or rate limiting configured
Files: */main.ts for all 3 APIs
// CURRENT (no security): const app = await NestFactory.create(AppModule); const port = process.env.PORT || 3000; await app.listen(port); // REQUIRED FOR PRODUCTION: import * as helmet from 'helmet'; const app = await NestFactory.create(AppModule); app.use(helmet.default()); app.enableCors({ origin: ['https://ethioconnect.com'] }); app.useGlobalPipes(new ValidationPipe({ whitelist: true })); await app.listen(port);

Impact: APIs accept requests from any origin (CORS wide open), no security headers (Helmet), no input validation, no rate limiting. Critical for production.

4.2 Critical: Environment Variables Not Validated
CRITICAL — No validation of required env vars (DATABASE_URL, JWT_SECRET, etc.)

APIs read process.env.PORT without validation. Missing: JWT_SECRET, DATABASE_URL, API keys. No schema validation (zod, joi, class-validator).

4.3 High: Secrets in Workflow Files
HIGH — Server credentials referenced in plain text in workflows
Files: ci.yml, release.yml, promote.yml, rollback.yml
// EXPOSED (ci.yml): SERVER_DOMAIN: ${{ secrets.SERVER_DOMAIN }} SERVER_USER: ${{ secrets.SERVER_USER }} SERVER_PASSWORD: ${{ secrets.SERVER_PASSWORD }} // Password-based SSH! SERVER_SSH_KEY_PATH: "" SERVER_SSH_PORT: ${{ secrets.SERVER_SSH_PORT || '22' }}

Issues: (1) Password-based SSH instead of key-based auth; (2) SSH key path is empty string; (3) No secret rotation strategy.

4.4 High: Dependency Confusion Risk
HIGH — Private packages use @ethio-connect scope without registry lock

Workspace uses @ethio-connect/* packages. Without .npmrc or pnpm.nexus.yml restricting the scope to a private registry, malicious packages could be injected via dependency confusion attacks.

4.5 Medium: No Input Validation on API Endpoints
MEDIUM — Controllers have no DTOs, no validation
// CURRENT (no validation): @Controller() export class AppController { @Get() getData() { return this.appService.getData(); } } // REQUIRED: @Controller() export class AppController { @Get() getData(@Query() query: GetDataDto) { ... } }
4.6 Medium: Next.js Apps No Auth Guards
MEDIUM — No authentication/authorization in Next.js pages

Dashboard pages have no auth checks, middleware, or route guards. Any unauthenticated user can access all pages.
